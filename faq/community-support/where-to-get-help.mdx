# Where can I get further support or report issues?

## Reporting Issues and Feature Requests

If you encounter a bug or have an idea to enhance go-pg, contribute by reporting it on the official issue tracker:

- **GitHub Repository:** [https://github.com/go-pg/pg/issues](https://github.com/go-pg/pg/issues)

### How to Report an Issue Effectively

1. **Search Existing Issues** to check if the problem or feature has already been reported.
2. **Provide a Clear Title** summarizing the problem or suggestion.
3. **Describe the Problem:**
   - What you expected to happen
   - What actually happened
   - Steps to reproduce
4. **Include Environment Details:**
   - Go version
   - go-pg version
   - PostgreSQL version
5. **Include Minimal, Complete Code Examples** that reproduce the issue when possible.
6. **Attach Error Messages or Logs** if applicable.

### Feature Requests

If your report is a feature request:

- Clearly explain the desired functionality
- Share the use case or rationale
- Discuss the possible benefits or scenarios

Maintainers review and prioritize requests based on community value and project roadmap.
